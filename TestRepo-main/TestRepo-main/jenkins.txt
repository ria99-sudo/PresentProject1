Maven Java Automation — Steps

Step 1: Open Jenkins (http://localhost:8080
)
→ Click New Item

Step 2: Create Freestyle Project (MavenJava_Build)
→ Enter project name
→ Click OK
→ Configure:

Description: Java Build demo

Source Code Management (SCM):

Git repository URL: [GitMavenJava repo URL]

Branch: */main or */master

Build Steps:

Add Build Step → Invoke top-level Maven targets

Maven version: MAVEN_HOME

Goals: clean

Add Build Step → Invoke top-level Maven targets

Maven version: MAVEN_HOME

Goals: install

Post-build Actions:

Add Post Build Action → Archive the artifacts

Files to archive: **/*

Add Post Build Action → Build other projects

Projects to build: MavenJava_Test

Trigger: Only if build is stable
→ Apply & Save

Step 3: Create Freestyle Project (MavenJava_Test)
→ Enter project name
→ Click OK
→ Configure:

Description: Test demo

Build Environment:

Check: Delete workspace before build starts

Add Build Step → Copy artifacts from another project:

Project name: MavenJava_Build

Build: Stable build only

Artifacts to copy: **/*

Add Build Step → Invoke top-level Maven targets:

Maven version: MAVEN_HOME

Goals: test

Post-build Actions:

Add Post Build Action → Archive the artifacts

Files to archive: **/*
→ Apply & Save

Step 4: Create Pipeline View for Maven Java
→ Click + beside "All"
→ Name: MavenJava_Pipeline
→ Select: Build pipeline view
→ Pipeline Flow:

Layout: Based on upstream/downstream

Initial job: MavenJava_Build
→ Apply & Save

Step 5: Run the Pipeline
→ Click RUN on pipeline
→ Check console output
→ Pipeline turns Green if successful
→ MavenJava_Test runs automatically after MavenJava_Build

→ Name: MavenWeb_Pipeline

→ Check console

→ Open Tomcat (localhost:8085)
→ Click /webpath to view deployed application
